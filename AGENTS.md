# Beerit Project Instructions

- Use TypeScript everywhere.
- Keep the app mobile-first.
- Use Tailwind CSS and shadcn/ui primitives for the interface.
- Use Supabase for authentication, Postgres data storage, and realtime updates.
- Keep a local demo fallback so the main flow can be tested before Supabase keys are configured.
- Use Beerits only as fictional in-game penalty points.
- Use Tokens only as creator reward points with no real-world value.
- Do not implement gambling, alcohol redemption, real-money rewards, or creator payouts.
- Keep responsible-play language visible in the product.
- Run lint and build checks before finalizing changes.
